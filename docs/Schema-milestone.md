# Schema sull'utilizzo delle Milestone di GitHub

## 1. Cos'è una Milestone
- Una milestone rappresenta un obiettivo o una fase del progetto.
- Serve per raggruppare issue e pull request legati allo stesso traguardo.

## 2. Quando usare una Milestone
- Per pianificare rilasci (release).
- Per organizzare attività in sprint o fasi di progetto.
- Per monitorare l'avanzamento di funzionalità complesse.

## 3. Creare una Milestone
1. Vai nella sezione **Issues** o **Pull requests**.
2. Clicca su **Milestones**.
3. Premi **New milestone**.
4. Compila:
   - **Title**: nome descrittivo.
   - **Description**: scopo della milestone.
   - **Due date** (opzionale): data prevista di completamento.
5. Salva.

## 4. Assegnare issue e PR a una Milestone
- Apri l'issue o la pull request.
- Nel pannello laterale, seleziona la milestone da assegnare.

## 5. Monitorare lo stato della Milestone
- Ogni milestone mostra percentuale di completamento.
- Barra di avanzamento basata sul numero di issue chiuse vs aperte.
- È possibile vedere elenco completo di issue e PR collegate.

## 6. Strategie di utilizzo
### 6.1 Milestone come release
- Ogni milestone corrisponde a una versione del software.
- Le issue rappresentano le funzionalità o fix da includere.

### 6.2 Milestone come sprint
- In contesti Agile, ogni sprint ha una milestone dedicata.
- Consente di visualizzare rapidamente lo stato dello sprint.

### 6.3 Milestone tematiche
- Raggruppano attività legate a una stessa feature.

## 7. Buone pratiche
- Usa descrizioni chiare e concise.
- Imposta una data di scadenza realistica.
- Mantieni aggiornata la milestone man mano che issue e PR vengono chiuse.
- Chiudi la milestone solo quando tutte le attività sono completate.

## 8. Esempio di struttura milestone
- **Titolo**: "Release 1.0"
- **Descrizione**: Stabilizzazione e primo rilascio pubblico.
- **Issue collegate**:
  - Implementazione feature A
  - Correzione bug critici
  - Documentazione finale

## 9. Conclusione
Le milestone di GitHub sono uno strumento potente per pianificare, organizzare e monitorare il lavoro. Utilizzarle correttamente permette una gestione più chiara del progetto e una migliore comunicazione nel team.
