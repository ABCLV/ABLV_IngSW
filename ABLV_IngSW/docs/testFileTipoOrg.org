* Chiavi Org-mode che val la pena provare
  ** Tab / Shift-Tab
     comprimi/espandi outline (molto utile su file lunghi)
  ** Alt-Left / Alt-Right
     sposta una voce (con tutto il sotto-albero) su/giù
  ** C-c C-t
     cicla TODO -> DONE -> nil su una voce
  ** C-c '
     edita il blocco di codio in un buffer separato (major-mode appropriato)
  ** C-c C-e h o
     esporta in HTML e lo apre nel browser (fatto in 1 s)

* Org vs Markdown: pro e contro rapidi
  *** Org
      + outline *veramente* gerarchica (ogni voce è un nodo)
      + checkbox TODO/DONE con statistiche automatiche ([2/4] diventa [3/4])
      + blocco di codio con sessioni live (literate programming)
      + esportazione nativa in PDF, ODT, Beamer, etc. senza toolchain esterne
      - si apprezza solo dentro Emacs (fuori è solo testo)
      - meno renderer “embedded” (GitHub lo visualizza, ma meno “pretty” di .md)

  *** Markdown
      + leggibile ovunque: GitHub, VS Code, Jupyter, etc.
      + sintassi minima, si scrive più rapidamente
      + estensioni (GFM) per tabelle, todolist, diagrammi
      - meno potente nell’outline (è solo ## ###)
      - le checkbox sono “finte” (non aggiornano contatori automatici)
      - per PDF serve Pandoc o Action esterna

* Conclusione di prova
  Se lavoriamo tutti su Emacs → Org è superiore.
  Se vogliamo la massima compatibilità → Markdown vince.
  In entrambi i casi, ricordiamoci: *una frase per riga* per evitare conflitti!
